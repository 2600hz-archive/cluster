Welcome to the VoIP Cluster Manager!

This tool is designed to help you deploy and manage one or more clusters of VoIP servers.
 
When we say "deploy", we're referring to the automatic installation, configuration and clustering of:
  • VoIP Switching Software (like FreeSWITCH)
  • Session Border Controllers / Load Balancers (like OpenSIPs)
  • Individual Session Control Tools (like eCallMgr)
  • Dialplan and User Authentication Systems (like Whistle's Registrar App)
  • Database Software (like BigCouch)
  • Best-Practice Firewall Settings (IPTables, fail2ban)
  • Monitoring Tools
 
On the next screen you'll be able to add your first server or set of servers. You'll be asked to make a decision. You can:
  • Deploy a single server with all components (useful for testing) with no ability to expand
  • Deploy a full suite of production servers (with the ability to expand the cluster later as you need)
 
Whichever option you choose, you'll gain access to the full suite of services documented at http://wiki.2600hz.org/ including our completely API driven PBX and VoIP call handling services.
 
WARNING:
The deployment requires your servers to be setup with CentOS 5.5 or 5.6. You must also properly match the hostname configured on the server with the hostname you enter in this tool (even if DNS isn't setup). Finally, the system clock must be accurate on the server you are deploying.
 
NOTE:
You don't need your own cluster to take advantage of 2600hz's services or take things for a test spin. Your account is already configured to use our demo server. When you're ready to move to production, we can host your service or you can. Contact us for more details at info@2600hz.com
